About My Cart4U website

About the application:

Outline:

  • Cart4U website is a basic E-Commerce webpage, used to view products in the home page by login into the website.

  • So that you can view and add the products to the cart.

  • If you are a new user , you can register using signUp page and you can login to do the operations

  • Here, users are stored in local storage.If you are new user while registering your details will get store in local storage.

Technology used:
  • HTML 5
  • CSS3
  • BootStrap v-5.3
  • Angular Js 16
Guard Created:

Auth Guard-used to do authentication

Components Created:
  • NavBar Component- used to do navigation operations to the particular page.
  • Footer Component-used to display footer section to UserInterface.
  • Login Component-used to login existing user.
  • Register Component-used to register as new user.
  • Home Component-used to display home page for the user.
  • Product Component-used to display products in the home page.
  • Orders Component-used to display orders detail in the orders page.
Services Created:
  • Auth Service-used to check is the user is logged in.
  • Cart Service-used to cart page methods.
  • Product Service-used to Product page methods.
  • Storage Service-used to Storage page methods.
Steps used and challenges faced to build this application:-

  1. This Project consists of multiple components such as Home, Cart, Login, Register ,product and for the navigation
    between components Routing has been used via app routing module by providing respective routes in the route array.

  2. The respective service and models required for the application has been generated using the commands in git bash

  3. Used Template Driven form validation for the login and register page.

  4. For fetching the products an Api call has been made using HTTPClient (GET) request.

  5. Stored the product, cart and order details in the Local Storage . Stored the user details in the Local Storage.

  6. The add to cart and orders functionality logic has been designed in a logic that each user can access only his/her
    cart and order details in his/her login.

  7. No logged in user can access cart order and log out so for that CanActivate() method of auth guard in angular.
    For this process local storage has been used.

  8. Every Authenticated user can access cart to add or remove product also check out product to see it the orders page.

Challenges Faced:

Frequent API call - Used Observer() method to reduce the API call by fetching and checking it with the local storage.
Handling Cart for individual user.
